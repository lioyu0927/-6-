> You are an expert audio software engineer and Python developer specializing in real-time computer vision (MediaPipe/OpenCV) and digital signal processing (DSP) using PyAudio/SoundDevice.
>
> Your task is to generate a complete, production-ready single-file Python application (or a clean modular structure if necessary) for an interactive, touchless audio workstation.
>
> The application must implement two primary modes:
> 1. Audio Effector Mode (Core Backbone)
> 2. Virtual Synthesizer Mode (Extended Mode)
>
> **[Technical Specifications & Requirements]**
>
> 1. Real-time Audio I/O (Blackhole 2ch Integration)
> - Audio input and output must be handled via 'Blackhole 2ch' to capture the system audio (macOS/Windows virtual routing) and output the processed signal in real-time.
> - Configure block size (latency-optimized) and sample rate (44.1kHz or 48kHz) properly for live performance.
>
> 2. Computer Vision & Gesture Mapping (MediaPipe)
> - Use MediaPipe Hands to track 2D/3D coordinates of hand landmarks via the webcam.
> - Implement smooth interpolation (e.g., Exponential Moving Average) for the coordinates to prevent audio glitches/zipper noise during sudden hand movements.
>
> 3. Mode 1: Audio Effector Mode (Backbone)
> - Route the system audio input through a DSP chain including:
>   - EQ Filter (Low-pass / High-pass / Band-pass)
>   - Flanger (with LFO depth/rate control)
>   - Reverb (Wet/Dry mix, decay time)
> - Map specific hand gestures or positions (e.g., Hand 1 X/Y coordinates, distance between thumb and index finger) to control these DSP parameters in real-time.
>
> 4. Mode 2: Virtual Synthesizer Mode
> - When toggled, activate an internal Sound Generator alongside the Effector chain.
> - Include:
>   - Oscillator (Sine, Square, Sawtooth wave generation).
>   - Arpeggiator Module (automatically steps through a sequence of notes based on the hand's vertical pitch/grid position or finger counting).
> - The generated synth sound must also pass through the Effector chain defined in Mode 1.
>
> **[UX & Design Philosophies to Reflect in Code]**
> - Educational Accessibility: The control mapping should be intuitive enough for children to understand the relationship between physical movement, pitch, and sound modification.
> - Touchless Interface: Eliminate physical barriers; all interactions (switching modes, triggers, parameter tweaks) must be done solely via the camera.
> - Live Performance Reliability: Ensure lightweight, non-blocking threading so that the audio stream does not stutter while processing video frames.
>
> **[Output Format]**
> Provide a fully functional, well-commented Python file layout. Include all required imports, class structures (`AudioEngine`, `GestureTracker`, `AppController`), and clear instructions on how to install the dependencies (`pip install ...`).
> Do not omit the core DSP processing logic; implement the math/filters cleanly using NumPy.
